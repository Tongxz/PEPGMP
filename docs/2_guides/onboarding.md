# 新成员入职指南 (Onboarding Guide)

欢迎加入“人体行为检测系统”项目团队！我们很高兴有你的加入。

本指南旨在为你提供一条清晰、高效的路径，帮助你在第一周内快速熟悉项目、搭建好开发环境，并成功提交你的第一个贡献。

---

## 阶段一：你的第一天 —— 建立宏观认知

**目标**: 理解项目是做什么的、它的架构是怎样的，以及我们如何组织文档。

- **[ ] 任务1：阅读项目概览 (约15分钟)**
  请从项目根目录的 **[`README.md`](../README.md)** 文件开始。它会告诉你项目的核心价值、主要功能和技术栈。

- **[ ] 任务2：探索知识库地图 (约20分钟)**
  打开我们全新的 **[`知识库主页 (INDEX.md)`](../INDEX.md)**。这个文件是我们所有文档的导航中心，请花些时间熟悉我们是如何组织架构、指南和参考手册的。

- **[ ] 任务3：理解核心架构 (约25分钟)**
  请阅读 **[`系统架构设计文档`](../1_architecture/system_design.md)**。你不需要理解每一个细节，但请重点关注以下几点：
    - 系统是如何通过 **Redis** 进行实时通信的。
    - **调度器 (Scheduler)** 和 **执行器 (Executor)** 的分层设计理念。
    - 了解“检测进程”和“API服务”是相互分离的。

---

## 阶段二：你的第二天 —— 搭建开发环境

**目标**: 在你的本地机器上成功运行项目并跑通所有测试。

- **[ ] 任务1：遵循贡献者指南搭建环境 (约1.5小时)**
  这是最关键的一步。请打开根目录的 **[`CONTRIBUTING.md`](../CONTRIBUTING.md)** 文件，并严格遵循 **“开发环境搭建”** 一节的指引完成以下操作：
    1.  克隆项目代码。
    2.  创建并激活Python虚拟环境。
    3.  安装所有开发依赖 (`pip install -e .[dev]`)。
    4.  **安装 pre-commit 钩子 (`pre-commit install`)**。这一步至关重要，它能确保你的代码在提交时自动符合项目规范。

- **[ ] 任务2：验证你的环境 (约10分钟)**
  环境搭建好后，运行完整的测试套件来验证一切是否正常：
  ```bash
  pytest
  ```
  如果所有测试都显示 `PASSED`，那么你的环境就准备好了！

---

## 阶段三：你的第一周 —— 提交第一个贡献

**目标**: 完成一个真实的小任务，并成功将代码合入 `develop` 分支。

- **[ ] 任务1：认领你的第一个任务**
  - 与你的导师或团队负责人沟通，他们会为你分配一个适合上手的任务。
  - 你也可以主动在项目的 `Issues` 页面寻找带有 `good first issue` 标签的任务。

- **[ ] 任务2：遵循Git工作流进行开发**
  - 再次参考 **[`CONTRIBUTING.md`](../CONTRIBUTING.md)** 中的 **“Git 工作流”** 部分。
  - 从 `develop` 分支创建一个属于你的 `feature/...` 分支。

- **[ ] 任务3：编写代码与测试**
  - 专注完成你的任务。请记住，所有新添加的逻辑都应该有相应的单元测试来覆盖。

- **[ ] 任务4：提交代码并创建Pull Request (PR)**
  - 当你完成开发和测试后，提交你的代码。`pre-commit` 会在此时检查你的代码风格和质量。
  - 遵循 **“Commit Message 规范”** 编写有意义的提交信息。
  - 在 GitHub 上创建一个指向 `develop` 分支的 Pull Request。

- **[ ] 任务5：参与代码审查 (Code Review)**
  - 你的PR提交后，CI/CD流水线会自动运行所有检查。同时，团队成员会对你的代码进行审查。
  - 请积极回应审查中提出的评论和建议。这是一个非常好的学习机会。
  - 当所有检查通过且获得批准后，你的代码就会被合并。恭喜你！
