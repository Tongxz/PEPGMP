# 视频流配置说明

## 当前配置参数说明

### 1. `log_interval` (检测间隔)
- **作用**: 控制检测的频率
- **说明**: 每N帧执行一次检测
- **示例**: `log_interval=120` 表示每120帧检测一次
- **影响**: 
  - 检测频率越低，CPU占用越低
  - 检测频率越高，检测结果越实时

### 2. `stream_interval` (视频流推送间隔)
- **作用**: 控制视频流推送的频率
- **说明**: 每N帧推送一次到视频流
- **示例**: `stream_interval=3` 表示每3帧推送一次
- **影响**:
  - 推送频率越低，带宽占用越低
  - 推送频率越高，视频流越流畅

### 3. `frame_by_frame` (逐帧模式)
- **作用**: 是否逐帧推送视频流
- **说明**: 开启后，`stream_interval` 自动设置为1（每帧都推送）
- **影响**: 最高帧率，但带宽占用最大

## 当前逻辑

### 检测时
- 如果 `frame_count % log_interval == 0`，执行检测
- 如果同时 `frame_count % stream_interval == 0`，推送检测后的帧（带标注）

### 不检测时
- 如果 `frame_count % stream_interval == 0`，推送原始帧（不检测，不带标注）

## 问题

当前 `log_interval` 和 `stream_interval` 是独立的，可能导致：
- 检测频率和视频流推送频率不一致
- 视频流可能显示未检测的原始帧

## 用户需求

用户希望：
1. 在配置相机时，可以配置相机的检测频率（`log_interval`）
2. 视频流的实时显示功能直接使用相机配置时的检测频率来进行显示

这意味着：
- 视频流推送应该和检测频率保持一致
- 视频流应该显示检测后的结果（带标注的帧）
- 可以简化配置，只需要一个检测频率参数
